# 設定頁手動驗收清單（UAT Checklist）

> 對象：設定 ⚙️ 面板（後端 Provider／模型／API Key／GitHub／並行 等欄位、儲存、重新部署、改密碼）
> 用法：由真人逐列操作並填寫「實際結果」與「Pass/Fail」。每一列只驗一個點。

---

## 前置條件

驗收前請完成下列步驟，全部 OK 才開始逐項測試：

1. 在專案根目錄啟動伺服器：
   ```
   .venv/bin/python3 -m studio.server
   ```
2. 開啟瀏覽器，進入 <http://localhost:8000>。
3. 完成登入（如系統要求帳號／密碼）。
4. 點右上角 **⚙️ 設定** 按鈕，開啟設定面板。
5. 確認面板內欄位依分組顯示：**一般 / Claude / OpenAI / GitHub / 並行**，下方另有 **重新部署** 與 **變更密碼** 區。

> 備註：設定會寫入伺服器的 `.env`，儲存後下次討論即生效。秘密欄位（API Key／Token）只顯示「是否已設定」，不會顯示明文。

## 表格欄位說明

各區塊一律使用下列固定欄位，**每列僅一個驗證點**：

| 欄位 | 說明 |
| --- | --- |
| # | 案例編號 |
| 功能區塊 | 此案例所屬功能分區 |
| 操作步驟 | 真人實際要做的動作（白話、可照做） |
| 預期結果 | 具體可觀察的畫面或行為 |
| 實際結果 | 測試當下實際看到的結果（測試時填寫） |
| Pass/Fail | 通過填 Pass，未通過填 Fail（測試時填寫） |
| 備註 | 缺陷單號、截圖、補充說明 |

## 填寫說明

- 逐列照「操作步驟」操作，把所見填入「實際結果」，並在「Pass/Fail」標記 Pass 或 Fail。
- 與「預期結果」不符即填 Fail，於「備註」記下缺陷單號或截圖連結。
- 一列只驗一個點；若一步驟牽涉多個檢查點，請拆成多列。
